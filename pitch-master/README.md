# MEETENGINE — МАСТЕР-ПИТЧ (дек-склад)

Единый источник презентации. Большой веб-дек, собранный из данных. Из него режутся клиентские срезы.
**Дек НИКОГДА не редактируется руками в HTML — только через `data/*.json`, потом пересборка.**

## Что где лежит
```
pitch-master/
  index.html        # СОБРАННЫЙ мастер-дек (артефакт build). Открыть в браузере — работает.
  template.html     # шаблон: вся вёрстка + рендер-логика. Данные вставляются в __DECK_DATA__.
  build.js          # сборщик (Node): data → index.html и срезы в /cuts.
  data/
    meta.json       # default_anchor (100k), контакт, суть, дисклеймеры, open_questions
    cases.json      # банк кейсов (4 хиро + грид + PunchR), теги segment/angle/objection_ids
    brands.json     # табло (9 подтверждённых) + wall (флаг «подтвердить») + inside_proof
    niches.json     # сегменты + оффер-варианты + список «что остаётся у клиента»
    prices.json     # ценовая лестница (склад) + закрытия-якоря + калькулятор окупаемости
    objections.json # спина деки: 17 ступеней, каждая с attach (что рендерится после)
    founder.json    # видео основателя, отстройка, миссия (деловым языком)
  assets/           # реальные лого + founder.mp4
  cuts/             # сюда build.js кладёт клиентские срезы
```

## Как собрать
Мастер-дек (после правки данных):
```
node build.js
```
Клиентский срез (мастер не трогается):
```
node build.js --client="ADVANZA" \
  --segments=event \
  --angles=высота-ЛПР,охват \
  --objections=1,2,6,9,14,15,17
```
→ `cuts/pitch-advanza/index.html` — только релевантные кейсы, ниши и ступени, с титулом под клиента.

Фильтры: `--segments` и `--angles` фильтруют кейсы и ниши (кейс остаётся, если совпал сегмент ИЛИ угол); `--objections` оставляет только перечисленные ступени. Без флага — берётся всё.

> В окружении без Node мастер-`index.html` бутстрапится тем же данным через Python (см. историю сборки). Канонический сборщик — `build.js`.

## Теги (для срезов)
- **segment:** digital · performance · influence · pr · smm · creative · event · production · webdev · adtech · it-consult · other
- **angle** (что доказывает кейс): охват · скорость-денег · высота-ЛПР · сложность-захода · масштаб-встреч
- **objection_id:** 1–17 (см. objections.json)

## ПРОТОКОЛ НАКОПЛЕНИЯ (обязательно)
Новый кейс / встреча / бренд / возражение / цена:
1. Добавить объект в нужный `data/*.json` **с тегами** (segment/angle/objection_ids). Ничего не писать руками в HTML.
2. `node build.js` — пересобирает мастер-дек и (при желании) активные срезы.
3. Строка в `CHANGELOG.md` (дата, что добавлено).
4. Если это новый факт для проекта — дописать в `ПРОЕКТ-ПАМЯТЬ_MASTER`.

## ЗАКОН ЧЕСТНОСТИ
- Только реальные кейсы/цифры. Каждое лого = реальная Zoom-встреча; нет подтверждения — нет лого.
- ADVANZA — не кейс внедрения, в пруфы не входит.
- `[Ф]` факт с источника · `[Г]` гипотеза · `⟨уточнить⟩` плейсхолдер. Пустое честнее выдуманного.
- Источник правды приоритетнее этого README: `ПРОЕКТ-ПАМЯТЬ_MASTER` (последняя версия).
